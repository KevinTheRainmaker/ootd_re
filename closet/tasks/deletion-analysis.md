# 삭제 분석 — CLOSET Flutter MVP

## 분석 기준

1인 개발자, Flutter, MVP 검증 목적. "지금 검증에 필요한가?"가 판단 기준.

---

## 제외 확정 (Won't — MVP에서 구현 안 함)

| 항목                                  | 이유                                                                          |
| ------------------------------------- | ----------------------------------------------------------------------------- |
| 커뮤니티 피드 (SNS 스타일)            | 사용자 콘텐츠 모더레이션 + 알고리즘 필요. CPS 핵심 문제(기록 번거로움)와 무관 |
| 팔로우/팔로워 시스템                  | 소셜 그래프 관리는 MVP 검증 범위 외. 별도 서비스 수준의 작업량                |
| 좋아요/댓글 기능                      | 커뮤니티 피드 제외 시 불필요. 모더레이션 리소스 낭비                          |
| 아이템 재고 관리 / 옷장 분류 고도화   | MVP 핵심 플로우(업로드→카드)와 분리. 추후 검증된 수요 확인 후 추가            |
| 다중 AI 모델 비교 / 앙상블            | Gemini 2.0 Flash Vision 단일 모델로 MVP 충분. 복잡도 불필요하게 증가          |
| 푸시 알림 (날씨 알림, 코디 리마인더)  | 알림 인프라(APNs/FCM) 설정 공수 큼. MVP에서 유저 리텐션 검증 전 불필요        |
| 카드 소셜 공유 (인스타그램 직접 연동) | 딥링크 + 외부 앱 API 연동. 카드 생성 가치 검증 후 추가해도 충분               |
| 스타일 태그 / 해시태그 검색           | 검색 인덱스 구축 필요. 커뮤니티 없는 MVP에서 활용처 없음                      |
| 온보딩 튜토리얼 슬라이드              | MVP는 소수 테스터 대상. 직접 설명 가능한 단계에서 불필요한 개발 비용          |
| 다크모드 전환 UI 설정                 | Flutter ThemeMode는 시스템 설정 따르면 충분. 사용자 토글 UI 불필요            |

## 권고사항

1. **Android 우선 배포**: iOS 배포는 MVP 검증 후로 연기. Google Play 배포($25 1회성)가 Apple($99/년)보다 저렴.

2. **AI 분석 결과 즉시 저장**: Gemini Vision 호출 후 결과를 Supabase에 저장하여 재분석 방지. API 비용 통제의 핵심.

3. **Pro 기능은 UI만 선 구현**: remove.bg 연동은 실제 Pro 구독자가 발생할 때까지 지연. "Pro 전환 버튼"만 만들고 Stripe/인앱결제 연동은 검증 후 추가.

4. **커뮤니티 기능은 삭제가 아닌 연기**: CPS에 "커뮤니티에서 스타일 발견" 언급이 있으나, MVP에서는 제외. 개인 기록 가치 검증 후 Phase 2로 이동.

5. **S15 QA를 각 피처 완료 시마다 실시**: `flutter analyze` + `flutter build apk --debug`를 매 스프린트 완료 시 실행. 마지막 한꺼번에 몰리면 디버깅 비용 폭증.
