# CLOSET — TTH 사일로 공유 메모리

## Project Context
- CLOSET: Flutter 패션 앱 (iOS + Android)
- 핵심 플로우: 소셜 로그인 → 날씨+AI코디 홈 → OOTD 업로드+AI분석 → 카드 생성
- Stack: Flutter (Dart) + Riverpod 3.0 + Supabase + Gemini 2.0 Flash Vision + OpenWeatherMap
- 위치: C:\Users\HCIS\Desktop\git\ootd-re\closet\

## Key Decisions
- AI 패션 인식: Gemini 2.0 Flash Vision API (정확도 1위, 저렴)
- 날씨 API: OpenWeatherMap (무료 티어, Flutter 패키지 풍부)
- 상태관리: Riverpod 3.0 (1인 개발 표준, GetX 금지)
- 백엔드: Supabase (Storage + PostgreSQL, Flutter SDK 공식 지원)
- 배경제거: remove.bg API (월 50장 무료)
- Freemium: 무료(기본 카드) / Pro(AI 배경제거, 스타일 변경)

## Failure Lessons
- login_screen.dart처럼 다른 팀원 파일에 lint 경고가 생겨도 flutter analyze가 실패하면 직접 수정 필요 (파일 경계 엄격 적용은 기능 코드에 한함, lint는 전체 책임)
- router.dart를 덮어쓸 때 젠슨이 이미 auth redirect + refreshListenable 구현했으므로 보존 필수 (파일 경계 확인 우선)
- flutter analyze는 --no-pub 플래그 사용 필수 (pub resolution이 SDK 불호환 버전을 당겨올 수 있음)
- flutter build 전 flutter clean 시 build 디렉토리가 잠겨있으면 clean이 실패할 수 있음 — 재시도하면 해소됨
- NDK 버전 충돌: flutter.ndkVersion 대신 "27.0.12077973" 명시로 해결 (플러그인들이 27.x 요구)
- shared_preferences_android dependency_overrides 주의: 2.4.20은 Dart 3.9 요구 → 2.4.13으로 고정 필수 (SDK 3.7.2 환경)
