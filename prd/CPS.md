# CPS (Context-Problem-Solution)

## Context

**누가**: 1인 개발자가 사이드 프로젝트로 구축  
**어디서**: 웹 앱 (Next.js 기반, 브라우저 우선)  
**환경**: 패션에 관심 있는 MZ세대 사용자들이 인스타그램·블로그·틱톡 등 SNS 기반으로 착장을 콘텐츠화하는 문화가 이미 형성된 시장

## Problem

**핵심 문제**: 착장을 기록하고 싶지만 기존 방법(사진첩 저장, SNS 직접 글작성, 수동 아이템 정리)이 너무 번거로워 지속적 기록이 이루어지지 않음

**구체적 페인포인트**:

- 사진 찍기 → 아이템 목록 수동 작성 → 해시태그 직접 달기 → SNS 업로드까지 평균 10-15분 소요
- 크리에이터는 팔로워에게 착장 정보(브랜드, 가격, 구매처)를 알리기 위해 별도 게시물 작성 필요
- 과거 착장을 참고하고 싶어도 사진첩에 뒤섞여 찾기 어려움

**정량적 지표**: 미정 (MVP 검증 후 도출 예정)

## Solution

**해결 후 기대 상태**: 사진 한 장 업로드만으로 아이템 분리 → 설명 자동 작성 → OOTD 카드 자동 생성까지 30초 내 완료

**성공 지표 (Primary)**: D7 재방문율 40% 이상  
**성공 지표 (Secondary)**:

- 카드 저장/공유율 20% 이상
- 자동 생성 설명 수정 없이 수용률 60% 이상 (AI 품질 검증)
- 무료→유료 전환율 3% 이상 (장기 목표)
