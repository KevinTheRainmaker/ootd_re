# 사용자 이해

## 페르소나 우선순위

> Round 4 메타 정리에서 **크리에이터를 Primary User로 격상**. 일반 사용자/패션 관심자는 Secondary.

| 페르소나                         | 역할                         | 점유 추정 | MVP에서 중요도 |
| -------------------------------- | ---------------------------- | --------- | -------------- |
| 1. 패션 크리에이터 (Primary)     | 바이럴 루프 + 사용 사례 검증 | 5-10%     | Must           |
| 2. 패션 관심 MZ (Secondary)      | 핵심 사용자 풀               | 60-70%    | Must           |
| 3. 일반 기록형 사용자 (Tertiary) | 부가 사용자                  | 20-30%    | Should         |

## 페르소나 1 — 지은 (Primary): 인스타 5만 팔로워 패션 크리에이터

**한 줄**: "내 옷 어디꺼인지 알려달라는 DM/댓글에 매일 30개씩 답한다."

- **나이/직업**: 26세, 프리랜서 / 인스타 패션 크리에이터
- **현재 행동**:
  - 인스타 스토리에 OOTD 올림 → "이거 어디꺼요?" DM 폭주
  - 답장에 1시간씩 소요. 링크트리에 자주 가는 쇼핑몰 링크 박아두지만 이탈률 높음
  - 협찬 받은 옷도 별도 표기 필요 (광고 공정화법)
- **숨겨진 니즈**:
  - "한 번 답하면 끝나는 도구". 매번 같은 답 반복하기 싫음
  - 자기 스타일 정체성을 시각적으로 어필 가능한 카드
  - 협찬 vs 자기 옷 구분 자동화 (Phase 2)
- **전환 트리거**:
  - 다른 크리에이터가 우리 링크를 인스타 바이오에 박고 있는 걸 봄
  - 30초 안에 카드 + 링크 생성되는 데모 영상
- **이탈 트리거**:
  - 카드 생성 30초 넘어감 → 답답함
  - AI가 아이템을 잘못 분석하는데 수정도 번거로움
  - 무료 5회 한도 빨리 소진되고 유료 가치가 애매함

## 페르소나 2 — 민지 (Secondary): 패션 관심 대학생

**한 줄**: "옷 좋아하지만 인스타 올릴 만큼 부지런하진 않다."

- **나이/직업**: 22세, 대학생
- **현재 행동**:
  - 매일 거울 셀카 찍지만 90%는 갤러리에 묻힘
  - 가끔 친구에게 "오늘 코디 어때?" 카톡 → 사진만 보냄, 정보 없음
  - 자기가 산 옷 브랜드를 자주 까먹음
- **숨겨진 니즈**:
  - 노력 없이 자기 스타일 아카이브 구축
  - 친구에게 자랑할 때 멋있는 카드로 보여주기
  - "내가 이런 스타일이구나" 자기 인식
- **전환 트리거**:
  - 친구가 카톡으로 우리 카드 링크 공유함
  - 사진만 올리면 알아서 카드 만들어준다는 점
- **이탈 트리거**:
  - 가입 절차 복잡
  - 첫 카드 결과물이 기대 이하 (손글씨가 어색하거나 분석이 틀림)
  - 3-4일 안에 다시 들어올 이유가 없음

---

## 페르소나 3 — 수현 (Tertiary): 직장인 기록형

**한 줄**: "옷 산 거 까먹어서 같은 옷 두 번 사는 게 싫다."

- **나이/직업**: 31세, 직장인
- **현재 행동**:
  - 사진 갤러리에 폴더로 옷 관리 시도하다 포기
  - 노션/엑셀에 옷 리스트 만들었다 작심삼일
- **숨겨진 니즈**:
  - 자동 분류된 옷장 (브랜드, 색상, 카테고리)
  - 쇼핑 전 "내가 비슷한 거 있나" 확인
- **전환 트리거**:
  - AI가 자동으로 분류해준다는 점
  - 캘린더로 한 달 코디 한눈에 보기
- **이탈 트리거**:
  - 공유/카드보다 기록/검색이 약함 (우리 MVP는 기록 검색이 약함)
  - 캘린더가 단순 리스트뷰여서 만족도 낮음

## 말하는 것 vs 실제 행동

| 사용자가 말하는 것       | 실제 행동                             | 시사점                                                      |
| ------------------------ | ------------------------------------- | ----------------------------------------------------------- |
| "옷을 기록하고 싶어요"   | 앱 깔고 3일 후 안 들어옴              | 기록 자체가 동기가 아님. **공유 가능성**이 동기.            |
| "친구한테 보여줄 거예요" | 실제로는 본인 SNS에 올림              | 진짜 목표는 SNS 콘텐츠. 카드를 SNS 친화 포맷(1080x1920)으로 |
| "AI가 정확하면 좋겠어요" | 70% 정확도여도 사용함 (수정만 쉬우면) | 정확도보다 수정 UX가 중요                                   |
| "유료 결제 안 할 거예요" | 무료 5회 다 쓰면 결제 고민 시작       | 한도 도달 시점이 결제 의향 최고점. 그때 UX 푸시 필요        |
| "심플한 디자인 좋아요"   | 화려한 손글씨 카드를 선택함           | 말하는 취향과 실제 선호가 다름. A/B 테스트 필수             |
| "프라이버시 중요해요"    | 공유 링크는 누구나 보게 함            | 공개/비공개 기본값은 비공개, 명시적 토글 필요               |

## 문제 정의

### 표면적 문제 vs 근본 문제

- **사용자가 말하는 문제**:

  > "착장 기록하기 번거로워요. 사진 찍고 태깅하고 업로드하는 게 10-15분 걸려요."

- **실제 근본 원인**:

  > **공유할 가치 있는 콘텐츠 형태가 없음.** 일반 사진은 정보 부재 → "이 옷 어디꺼?" 페인 발생. 정보 카드를 만들려면 수동 작업 10분+. 그래서 기록 자체를 포기.

- **왜 지금인가**:
  1. AI Vision API의 패션 인식 정확도가 2024년 이후 실용 수준 도달
  2. gpt-image-1 같은 이미지 편집 AI 상용화
  3. 스타일쉐어 폐쇄 후 국내 OOTD 공백 2년+ 누적
  4. 크리에이터 경제 성숙, 광고 공시 의무 강화

### 사용자가 "기록"이라 말하지만 진짜 원하는 것

| 표면 (말) | 심층 (실제 원함)          |
| --------- | ------------------------- |
| 기록      | 자기 정체성의 시각적 증명 |
| 검색      | 동일 코디 회피 (창피함)   |
| 공유      | 칭찬, 인정                |
| 분석      | 본인 스타일 이해          |

→ **MVP는 "시각적 증명" + "공유"에 집중**. "기록/검색/분석"은 부가.
