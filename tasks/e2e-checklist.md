# E2E 체크리스트 — OOTD-re MVP

> Phase 4 Full Eval 진입 전 검증. 모든 항목 PASS 후 베타 배포.
> 환경: 로컬 `npm run dev` + `.env.local` 실 API 키 세팅 필요.

---

## 플로우 A — 핵심 가치 검증

### A1. 비로그인 홈 접근 → 로그인 유도

- [ ] `/` 접근 시 홈 페이지 정상 렌더링
- [ ] 헤더에 "시작하기" 버튼 표시 (로그인 버튼, 프로필 드롭다운 아님)
- [ ] "시작하기" 클릭 → `/api/auth/signin` 또는 `/auth/signin` 페이지 이동
- [ ] `/upload` 직접 접근 → 로그인 페이지로 리다이렉트 (middleware 동작)
- [ ] `/ootd` 직접 접근 → 로그인 페이지로 리다이렉트

### A2. Google 로그인 → 홈 이동

- [ ] 로그인 페이지에서 Google 버튼 클릭 → Google OAuth 팝업/리다이렉트
- [ ] 동의 후 홈(`/`)으로 리다이렉트
- [ ] 헤더에 프로필 이미지(또는 이니셜) 표시
- [ ] 프로필 드롭다운 클릭 → "내 OOTD", "캘린더", "로그아웃" 항목 확인
- [ ] Supabase `users` 테이블에 신규 row 생성 확인 (첫 로그인 시)

### A3. 사진 업로드 → AI 분석 결과 표시

- [ ] `/upload` 접근 정상
- [ ] 파일 선택 버튼으로 JPG/PNG/WebP 선택 가능
- [ ] 모바일: 카메라 직접 촬영 옵션 표시 (`accept="image/*" capture`)
- [ ] 선택 후 미리보기 표시
- [ ] "분석 시작" 버튼 클릭 → 로딩 상태 표시
- [ ] AI 분석 완료 → 아이템 목록(카테고리, 색상, 설명) 표시
- [ ] 스타일 요약 문장 표시
- [ ] 해시태그 목록 표시
- [ ] 분석 소요 시간 10초 이내 (P95 기준)

### A4. 아이템 편집 → 카드 생성

- [ ] 아이템 카드 클릭 → 편집 모드 진입
- [ ] 카테고리 드롭다운 변경 가능
- [ ] 색상 필드 수정 가능
- [ ] 설명 필드 수정 가능
- [ ] 편집 저장 → 목록 즉시 반영
- [ ] "카드 만들기" 버튼 클릭 → 카드 생성 로딩 (최대 30초)
- [ ] 카드 이미지 표시 (Plan A 또는 Plan B)
- [ ] Plan B fallback 시 안내 메시지 표시

### A5. 카드 저장

- [ ] `/card` 페이지에서 "저장하기" 버튼 클릭
- [ ] 로딩 스피너 표시 후 "저장 완료" 상태로 전환
- [ ] 성공 토스트 "카드가 저장됐습니다." 표시
- [ ] Supabase `ootd_records` 테이블에 row 생성 확인
- [ ] "PNG 저장" 버튼 → 로컬 파일 다운로드

### A6. 공개 설정 → /share/[id] 링크 복사 → 비로그인 접근

- [ ] 저장 페이지 또는 상세 페이지에서 공개 토글 ON
- [ ] 공개 전환 후 `share_id` 생성 확인 (DB + URL)
- [ ] "링크 복사" 버튼 클릭 → 클립보드에 `/share/[share_id]` URL 복사
- [ ] 성공 토스트 "링크가 복사됐습니다." 표시
- [ ] 다른 브라우저(비로그인) 에서 복사한 URL 접근
- [ ] 카드 이미지 + 아이템 목록 + 작성자 닉네임 표시
- [ ] OG 메타태그 확인 (브라우저 개발자도구 → `<meta property="og:image">`)
- [ ] "이 카드 만들기" CTA 버튼 표시

## 플로우 B — 제한 검증

### B1. 무료 사용자 카드 생성 5회 한도

- [ ] 카드 생성 1-5회: 정상 생성
- [ ] 헤더 또는 대시보드에서 "이번 달 N/5회 사용" 카운터 확인
- [ ] 6회째 "카드 만들기" 클릭 → 업그레이드 안내 모달 표시
- [ ] 모달에 현재 사용량(5/5), 한도 초과 메시지, 플랜 안내 포함
- [ ] 모달 닫기 후 카드 생성 불가 상태 유지
- [ ] 다음 달 기준(year_month 변경) → 카운터 리셋 확인 (lazy reset 동작)

### B2. 비공개 OOTD 공유 링크 차단

- [ ] 비공개(`is_public = false`) OOTD 생성
- [ ] 해당 OOTD의 share_id로 `/share/[id]` 직접 접근
- [ ] 404 페이지 반환 확인
- [ ] 본인 로그인 상태에서도 `/share/[id]`로는 404 (상세는 `/ootd/[id]`로만 접근)

## 플로우 C — 캘린더

### C1. 저장된 OOTD 캘린더 확인

- [ ] 헤더/하단 탭 "캘린더" 클릭 → `/calendar` 이동
- [ ] 현재 월 OOTD 리스트 표시 (썸네일 + 날짜 + 스타일 요약)
- [ ] 이전 월 버튼 → 이전 월 데이터로 전환
- [ ] 다음 월 버튼 → 다음 월 데이터로 전환
- [ ] 데이터 없는 월 → "이번 달 OOTD가 없습니다" 상태 표시

### C2. OOTD 상세 페이지

- [ ] 캘린더 항목 클릭 → `/ootd/[id]` 상세 페이지 이동
- [ ] 카드 이미지 + 아이템 목록 + 날짜 + 메모 표시
- [ ] 공개/비공개 상태 표시

### C3. 편집 및 삭제

- [ ] 상세 페이지에서 편집 버튼 클릭 → 편집 모드
- [ ] 메모 수정 → 저장 → 반영 확인
- [ ] 공개/비공개 토글 → DB 반영 확인
- [ ] 삭제 버튼 클릭 → 확인 모달 → 삭제 실행
- [ ] 삭제 후 캘린더 페이지로 리다이렉트 + 목록에서 제거 확인
- [ ] 삭제된 OOTD의 `/share/[id]` 접근 → 404
