# 삭제 분석 — OOTD-re MVP

## 분석 기준

- 1인 개발자, 1개월 타임라인 (Claude Code 활용)
- MVP 핵심 가치: 사진 → AI 분석 → 카드 → /share/ 링크. 그 외는 부가.
- 기준 질문: "이 기능 없이도 베타 5명이 크리에이터로서 인스타 바이오에 링크를 박을 수 있는가?"

## 권고사항 (사티아에게 전달할 핵심 결정 사항)

1. **Sprint 0 완료 전 S10 착수 금지**: gpt-image-1 PoC go/no-go 결과 없이 카드 생성 API를 짜면 Plan A/B 모두 짜게 됨. Sprint 0 = 블로킹 게이트.

2. **Plan B 명세 수정 필요**: SPEC.md의 "html2canvas (브라우저)" 표현을 **satori(@vercel/og)** 로 공식 교체. html2canvas는 서버에서 안 됨. 이 수정은 젠슨(Backend) 담당 파일(src/lib/ai/) 변경 없이 SPEC.md 수준 수정으로 가능.

3. **Kakao OAuth는 Week 3으로 밀기**: Google만 Week 1 출시. RISKS R10 완화. 일정 압박 시 Sprint 0 직후 바로 체감됨.

4. **ootd_items 테이블에서 제거할 컬럼**: `purchase_url`, `price`, `size`, `is_ad` — 4개. MVP 핵심 루프에 불필요, DB 마이그레이션에 포함 안 해도 됨. Phase 2에 ALTER TABLE로 추가 가능.

5. **분당 API Rate Limit는 MVP 제외**: usage_logs 월 한도만. Vercel 서버리스 동시성이 자연 제한 역할. 남용 발생 시 Upstash Redis로 추가.

6. **OG 이미지 전략 결정 필요**: 1080x1920 카드를 OG에 그대로 쓰면 카카오톡 미리보기 깨질 가능성. Week 3 M1.9 작업 시 satori로 1200x630 OG 전용 이미지 별도 생성 여부 결정 필요.
